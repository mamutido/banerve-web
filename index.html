How it works

Built for apps that
work without internet

BANERVE's SDK runs entirely on-device. No constant connection needed — ad data syncs once a day over WiFi.

01
📲
SDK integrates into your app
Publishers embed the lightweight SDK for Android, iOS, or watchOS (Apple Watch). It initialises with a unique app_id and downloads creatives + targeting rules in the background.
02
🧠
Targeting runs on device
Before every ad slot, the SDK checks signals locally — battery level, connected headphones, weather, language, time of day. No data is sent to any server. Privacy is structural, not policy.
03
📦
Impressions sync in batches
Impression counts accumulate on-device and sync once per day via WiFi. The server receives only anonymous counters — never user profiles, never location history.
04
⚡
Automatic financial settlement
Batch processing creates three ledger entries per ad pair: impression debit, payout to publisher, commission to platform. Transparent, immutable ledger.

{

Privacy by design

Zero data leaves
the device

🔒
On-device targeting
All 20+ targeting rules are evaluated locally, before each ad slot. The server never receives user profiles, location history, or behavioral data.
📊
Anonymous counters only
What reaches the server: a number. "Creative X was shown N times on date D in app Z." Nothing about who saw it or where they were.
📶
Offline-first SDK
Ad content and targeting rules are cached locally. The app works and shows ads without any active internet connection. Sync happens once per day via WiFi.
📋
Immutable financial ledger
Balances are never stored — they're always computed from the transaction log. Every financial event is a permanent, auditable record.
🛡
"Personal data never leaves the user's device. Ever."
This is structural — not a policy.
The server has no API to receive it.
